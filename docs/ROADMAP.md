# Roadmap — Toward "Open-Source OrCam-Class for $50"

> **North star**: a $50–100 BOM, open-source, DIY-buildable assistive vision
> device that does ~80% of what a $5,000 OrCam MyEye does — usable globally,
> especially in markets where commercial assistive technology is out of
> financial reach.

## Two tracks (locked in)

| Track | Audience | Backend | Status |
|---|---|---|---|
| **`v0.1.x` Pro** — full self-hosted | Hackers, privacy-first, content creators | Local Flask proxy → OpenAI / any provider | ✅ [v0.1.0 released](https://github.com/Techposts/tuya-t5-blind-navigator/releases/tag/v0.1.0) |
| **`v0.2.x` Easy** — DIY end-users | Mainstream builders, family of blind users | TuyaOpen + Tuya Cloud AI Agent (wake word built-in) | 🔄 In planning |

Both tracks share the same UI, gestures, sensors, enclosure, and language
packs — only the cloud backend differs. We build infrastructure once and
swap the backend.

## Where we are today (v0.1.x — Pro track shipped)

- ✅ Camera capture (480×480 @ 15 fps via GC2145)
- ✅ Bionic-eye LVGL UI with state-driven color
- ✅ Multi-gesture trigger map (single / double / long click + touchscreen tap)
- ✅ OpenAI `gpt-4o-mini` vision via local Flask proxy
- ✅ OpenAI `gpt-4o-mini-tts` audio (WAV)
- ✅ Persistent on-screen response panel
- ✅ Navigator-style prompts (path / where / action / why)
- ✅ GitHub repo + wiki + release with binary

### Known gaps blocking "DIY product" status

- ⚠ Wi-Fi + API key baked into firmware (every builder must rebuild from source)
- ⚠ No on-device config UI
- ⚠ No on-board safety layer (cloud round-trip 3–8 s — not real-time)
- ⚠ No battery / enclosure (still a dev board on a desk)
- ⚠ No fall detection, no instant haptic obstacle warning
- ⚠ No voice activation
- ⚠ Single language (English)

The phases below close every one of those gaps in priority order.

## Phase 1 — v0.2 Tuya Cloud "Easy" track (week 1)

**Goal**: 5-minute flash + mobile-app provisioning + wake word `Hey Tuya`.

| Task | Detail | Effort |
|---|---|---|
| Claim Tuya dev license | Per [Tuya's guide](https://tuyaopen.ai/docs/faqs/get-developer-license) — 2 free per account | 10 min (user) |
| Create new Tuya Cloud product | Smart speaker / AI agent category, T5AI module | 15 min (user) |
| Configure AI agent | System prompt (navigator), vision input enabled, speech output enabled, model = vision-capable LLM | 10 min (user) |
| Port firmware | New parallel app `apps/tuya.ai/blind_navigator_cloud/` using `tuya_iot_init` + `ai_chat_init(AI_CHAT_MODE_WAKEUP)` + `ai_video_init` | 1 day |
| Reuse bionic-eye UI | `nav_display.c` ports as-is — same exported function names | 0 (no change) |
| Multi-gesture dispatch | Use Tuya runtime prompt API to switch system prompts per gesture | 0.5 day |
| Mobile mini-app | Auto-generated by Tuya Cloud — configure caregiver panel | 1 hour |

**Success criteria**:
- Builder flashes one binary, scans QR via Tuya Smart app, says *"Hey Tuya, what's in front of me?"* and gets a spoken response within 8 s.
- Total setup time: under 10 minutes.
- Same `.bin` works for every builder (no source build required).

**Content angle**:
*"Building an AI vision assistant for blind users in 5 minutes — the easy way, with Tuya Cloud."*

## Phase 2 — Web UI + mDNS + captive portal (week 2)

**Goal**: device-as-server for ongoing config + status. Works for **both**
Pro and Easy tracks (built once, used by both).

| Task | Detail | Effort |
|---|---|---|
| Captive portal on first boot | Board creates `BlindNav-Setup` AP at `192.168.4.1`; user connects via phone, enters Wi-Fi creds + secrets | 2 days |
| mDNS service | Device announces as `blindnav.local` once on Wi-Fi | 0.5 day |
| HTTP config server | `libhttp` (already in TuyaOpen). Routes: `/` (status), `/config` (settings), `/preview` (live camera), `/history` (last queries), `/logs` (debug) | 2 days |
| Persist config | `tal_kv` (already wired in firmware) | 0.5 day |

**Success criteria**:
- Blank-flash a board, connect phone to `BlindNav-Setup`, fill Wi-Fi form, save → board joins home network.
- Browse to `http://blindnav.local` from any device on same Wi-Fi → see status dashboard, change volume, switch language, view recent queries.
- No source rebuild needed for any config change.

**Content angle**:
*"DIY accessibility device with a full web dashboard — config from your phone, no code."*

## Phase 3 — Sensor stack for actual safety (weeks 3–4)

**Goal**: instant collision warning + fall detection + person detection.
The cloud LLM remains the smart layer; sensors handle the safety layer.

| Sensor | Cost | Job |
|---|---|---|
| **LD2410C** 24 GHz mmWave | ~$10 | Detect humans (static + moving) 0.75–6 m, even through clothing |
| **VL53L1X** ToF laser | ~$7 | Precise mm-accuracy distance up to 4 m for walls / curbs / stairs |
| **LRA motor** + DRV2605L driver | ~$5 | Instant haptic warnings (intensity ∝ inverse-distance) |
| **BMI270** 9-axis IMU | ~$5 | Fall detection, step counter, compass |
| **Bone-conduction speaker** | ~$15 | Replaces stock speaker — preserves ambient sound (critical: blind users navigate by sound) |

**Software**:
- Independent **alerts thread** (small stack, high priority) running sensor fusion at 50 Hz
- Haptic patterns: short-short = "object near", continuous-pulse = "stop", long = "ready"
- Fall detection → cloud alert + emergency contact (caregiver app push)

**Success criteria**:
- Walk toward a wall → haptic buzz before any cloud query fires
- Stationary person 2 m ahead → buzz + (if asked) cloud describes them
- Sudden fall → cloud alert + audio prompt "are you OK?"

**Content angle**:
*"This is the layer OrCam can't beat with cloud-only — and we've added it for $40 in parts."*

## Phase 4 — Power + form factor (weeks 5–6)

**Goal**: wearable, all-day battery, durable enclosure.

| Item | Cost | Detail |
|---|---|---|
| 2000 mAh LiPo | ~$7 | 8+ hr active, ~24 hr idle |
| TP4056 USB-C charging IC | ~$1 | Standard charging IC |
| Power switch + boot button | ~$2 | External access |
| 3D-printed enclosure | ~$2 filament | Variants: lanyard, chest clip, glasses-mount adapter |

**Software**:
- T5 deep-sleep mode when idle 60 s
- Wake on KEY button or motion (via IMU)
- Battery level reported via mDNS web UI + voice alert at 15 / 5 %

**Success criteria**:
- 8+ hours active use, < 2 g/day idle drain
- Fits in shirt pocket or hangs as lanyard
- USB-C charge from any phone charger

**Content angle**:
*"From dev board to wearable — the enclosure is open source, here's the STL."*

## Phase 5 — Multi-language + polish (week 7)

**Goal**: usable in the markets that need this most. English-only is a tiny
addressable market; Hindi + Spanish + Mandarin + Arabic + Bengali multiplies
the reachable user base by 100×.

| Language | Priority | Notes |
|---|---|---|
| Hindi | HIGH | Indian market = millions of low-income blind users |
| Spanish | HIGH | LATAM coverage |
| Mandarin | MEDIUM | China + diaspora |
| Arabic | MEDIUM | MENA region, RTL UI considerations |
| Bengali | MEDIUM | Bangladesh + India |
| Portuguese | LOW | Brazil |
| French | LOW | West / Central Africa |

**Tasks**:
- Translate `NAV_*_PROMPT` macros (use the LLM itself for translation, then native-speaker review)
- Voice profile selector in web UI
- Auto-detect language from wake word inflection
- Cached earcons per language ("listening", "got it", "no answer")

**Success criteria**:
- Switch language in web UI → next query responds in that language
- Wake word listens for "Hey Tuya" + the localized variant

**Content angle**:
*"Open source means we can ship in any language. Here's it working in Hindi."*

## Phase 6 — Community + reach (ongoing)

**Goal**: scale beyond a single builder. The hardware is permissively open;
language packs, enclosure remixes, and feature contributions come from
others.

| Initiative | Detail |
|---|---|
| Caregiver mini-app | Tuya custom panel: device status, last queries, push messages, low-battery alerts |
| Language pack repo | Separate community repo for community-contributed translations |
| Documentation translations | Wiki + README in major languages |
| Maker / accessibility conferences | Demo at India Maker Faire, accessibility expos |
| Real-user pilot program | Partner with 1 NGO or accessibility org for 5–10 device pilot, collect feedback |
| YouTube series | Build series, deep-dives, language localizations, real-user testimonials |

**Success criteria** (12 months out):
- 100+ devices built by others outside this repo
- Language packs in 10+ languages contributed by community
- At least one published real-user testimonial / pilot result

## Honest framing for the channel

We will not market this as "a better OrCam." That's bait. The honest pitch:

> *"OrCam costs $5,000. I built something that does most of what it does
> for $50 — open source, anyone can build it, anyone can modify it,
> works in any language. Here's how, in collaboration with Tuya."*

That story is **truer**, **bigger** (addresses the 99% priced out, not the
1% with insurance), **more replicable**, and **aligned with the Tuya
partnership** (showcases the cloud + free dev license + hardware platform
all working together for real-world impact).

## Next concrete action

Phase 1 starts with **you** claiming the Tuya developer license. Once you
have `PRODUCT_ID + UUID + AUTHKEY` from the [TuyaOpen license walkthrough](https://tuyaopen.ai/docs/faqs/get-developer-license),
the firmware port can begin in parallel — most of the work (skeleton app,
copying bionic-eye UI, build glue) doesn't need the credentials, only the
final wiring.
